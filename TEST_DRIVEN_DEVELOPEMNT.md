Testing policy:

- TDD is the default.
- Start by writing a failing test.
- Implement the minimum required to make the test pass.
- Refactor only after tests are passing.
- Work in small, incremental steps.

If TDD is not appropriate:
- testing is still mandatory
- add the most suitable tests or verification for the implemented behaviour
- state why TDD was not used
- state what validation was performed

Completion standard:
- no work is complete until it has been tested or an explicit exception has been documented
